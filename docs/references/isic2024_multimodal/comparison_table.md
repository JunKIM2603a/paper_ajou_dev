## Strict-Compatible 해석

- 직접 비교 후보: `FusionNetX`, `Multimodal system for skin cancer detection`, `Dual-stage segmentation...`는 image(영상) + metadata(메타데이터) 조건과 비교적 가깝다.
- 부분 비교 후보: `Automated triage...`, `Hybrid Ensemble...`, `PMM-FL`은 patient context(환자 맥락), WB360 proprietary feature(독점 특징), diagnosis-informed relabeling(진단 정보 기반 재라벨링), custom dataset(맞춤 데이터셋) 등으로 인해 같은 protocol(실험 절차)에서만 비교해야 한다.
- feature-audit(특징 감사) 후보: `Explainable multimodal AI...`는 XAI 관점에서는 매우 유용하지만 `mel_thick_mm` 같은 pathology-derived feature(병리 유래 특징)가 등장하므로 strict baseline(엄격 기준 모델) 비교에서는 제외하거나 해당 feature 제거 후에만 비교해야 한다.
- 논문 주장에 사용할 때는 반드시 `patient-level split`(환자 단위 분할), `train-only preprocessing`(학습 데이터만으로 전처리), `threshold source`(임계값 출처), `metric definition`(지표 정의) 확인이 필요하다.
