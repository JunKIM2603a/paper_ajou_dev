## 1. Baseline 시험

### 1.1 방향

* 제일 Simple 한 것부터 시작해야함
    * 현재 코드는 `strict_main_input` tabular suite와 image backbone suite를 먼저 안정화하는 방향
    * multimodal은 tabular/image baseline protocol이 안정화된 뒤 `research.md` 기준 early/middle/late fusion 로드맵으로 구현

### 1.2 dataset

- [isic2024_strict_input_export.md](../eda/isic2024_strict_input_export.md)
    - strict model input: image + ordinary metadata
    - iddx_full sidecar: train-only privileged candidate
    - split: patient-level Triple Stratified Nested CV, 5 outer x 4 inner
    - export 단계: imputation/scaling/encoding 하지 않음
    - shared split artifact: `data/splits/isic2024_official_train_nested_5x4_seed42.csv`

### 1.3 Baseline

* Protocol sanity check
  * patient-level split audit
  * iddx_full / diagnosis field exclusion audit
  * train-only preprocessing audit
  * metric + validation-threshold audit
  * strict input export test와 tabular protocol test

* Single-modal baselines
  * metadata-only
    1. current suite: `xgboost`
    2. current suite: `catboost`
    3. current suite: `lightgbm`
    4. current suite: `ft_transformer`
    5. current suite: `ft_transformer_external`
    6. feature set: `strict_main_input`
    7. supported standalone but not current suite: `logistic_regression`, `svm`, `mlp`

  * image-only
    1. current suite: `resnet50`
    2. current suite: `efficientnetv2_s`
    3. current suite: `convnextv2_tiny`
    4. current suite: `eva02_s`
    5. current suite: `vit_b`
    6. current suite: `edgenext_s`
    7. input: lesion image only

* Multimodal baselines
  * metadata + image
    1. 현재 `run_multimodal_experiment.py`는 `NotImplementedError`를 내는 scaffold 상태
    2. 현재 paper-facing 실행 대상은 tabular-only와 image-only baseline
    3. `C-Early`: pooled image feature + `strict_main_input`/FT-Transformer tabular representation concat
       - 최소 neural fusion baseline
    4. `C-Middle`: cross-attention 또는 FiLM/GMU conditioning
       - 대표 deep fusion 비교군
    5. `C-Late`: image ensemble + tabular stack + OOF meta-learner
       - 가장 강한 실전 기준선
    6. 공통 규칙
       - OOF 없는 stacking 금지
       - patient-level nested CV 유지
       - `iddx_full` 또는 diagnosis text는 multimodal inference input으로 사용하지 않음

### 1.4 데이터 처리 흐름

1. raw data는 data/raw/isic_2024_challenge/에서 읽기만 한다.
2. `export_strict_input_dataset.py`로 strict input, iddx sidecar, nested split CSV를 생성한다.
3. tabular baseline은 `experiments/configs/dataset_specs/strict_main_input_v1.json`과 `experiments/configs/suites/tabular_baselines.json`을 기준으로 실행한다.
4. image baseline은 `experiments/configs/dataset_specs/image_preprocessed_v1.json`과 `experiments/configs/suites/image_baselines.json`을 기준으로 실행한다.
5. metadata 모델 학습 시 결측치/스케일링/인코딩은 fold train 안에서만 fit한다.
6. class weight, sampler, feature selection, calibration model도 fold train 또는 inner validation protocol 안에서만 산출한다.
7. threshold와 model choice는 inner validation에서만 선택한다.
8. `outer_test`는 최종 평가 전까지 model choice, threshold, calibration, feature selection에 쓰지 않는다.
9. multimodal fusion은 baseline 안정화 이후 같은 fold의 metadata row와 image output을 `isic_id`로 join하도록 구현한다.
10. `C-Late`는 tabular/image OOF 생성 -> meta-learner 학습 -> calibration -> `outer_test` final evaluation 순서로 실행한다.
11. multimodal calibration도 test fold가 아니라 train/inner-validation protocol 안에서만 fit한다.
