## 1. Baseline 시험

### 1.1 방향

* 제일 Simple 한 것부터 시작해야함
    * sklearn boost(meta), resnet(img) 모델 등 사용

### 1.2 dataset

- [isic2024_strict_input_export.md](../eda/isic2024_strict_input_export.md)
    - strict model input: image + ordinary metadata
    - iddx_full sidecar: train-only privileged candidate
    - split: patient-level locked holdout + 5-fold CV
    - export 단계: imputation/scaling/encoding 하지 않음

### 1.3 Baseline

* Protocol sanity check
  * patient-level split audit
  * iddx_full / diagnosis field exclusion audit
  * train-only preprocessing audit
  * metric + validation-threshold audit
  * dummy baselines

* Single-modal baselines
  * metadata-only
    1. metadata_dummy: DummyClassifier
    2. LogisticRegression
    3. LogisticRegression(class_weight="balanced")
    4. HistGradientBoostingClassifier
    5. HistGradientBoostingClassifier(class_weight="balanced")
    6. ExtraTreesClassifier(class_weight="balanced") optional
    7. LightGBM / XGBoost / CatBoost later strong tabular baselines

  * image-only
    1. image_dummy
    2. Resize + Flatten + PCA + LogisticRegression
       - sklearn-only smoke/sanity baseline
    3. frozen pretrained ResNet-50 embedding + LogisticRegression
       - simple but more meaningful image baseline
    4. fine-tuned ResNet-50
    5. fine-tuned DenseNet-121 / EfficientNet-B0
    6. ConvNeXt / ViT / DINOv2 later strong image baselines

* Multimodal baselines
  * metadata + image
    1. late fusion: metadata probability + image probability weighted average
    2. score stacking: [metadata probability, image probability] -> LogisticRegression
    3. embedding concat: [strict metadata features, frozen image embedding] -> LogisticRegression / MLP
    4. deep concat: image encoder feature + tabular encoder feature -> MLP head
    5. gated fusion / attention fusion later

### 1.4 데이터 처리 흐름

1. raw data는 data/raw/isic_2024_challenge/에서 읽기만 한다.
2. export_strict_input_dataset.py로 strict input, iddx sidecar, split CSV를 생성한다.
3. metadata 모델 학습 시 결측치/스케일링/인코딩은 fold train 안에서만 fit한다.
4. image 모델은 같은 locked split CSV를 사용한다.
5. multimodal fusion은 같은 fold의 metadata row와 image prediction/embedding을 isic_id로 join한다.
6. test_data는 최종 평가 전까지 model choice, threshold, calibration, feature selection에 쓰지 않는다.
